# Source priority

Use this order when scouting application evidence.

## 1. application note

If the user already has an application note, start there.

Look for:

- company
- role
- URL
- status
- existing resume or cover-letter references
- notes under why I want this, application materials, interview log, and decision

## 2. application tracker

Check `Areas/Career/Applications/_tracker.md` to understand pipeline state and whether the note is active, stale, or already closed.

## 3. work history notes

Use `Areas/Career/work-history-*.md` for stable career evidence and metrics.

## 4. resume framework

Use `Areas/Career/resume-framework.md` for preferred bullet structure and career framing.

## 5. project notes

Search `Areas/Projects/**` and nearby project notes for evidence that matches the JD.

Prefer projects with:

- direct technical overlap
- clear ownership
- measurable outcomes
- recent activity

## 6. active tasks and recent notes

Check `Inbox/*.md` and related notes for proof of recent work, current focus, and language that can support resume bullets or interview stories.

## ranking rules

Rank evidence by:

1. direct JD relevance
2. measurable impact
3. recency
4. ownership depth
5. interview usefulness

Prefer shortlists over exhaustive dumps.
