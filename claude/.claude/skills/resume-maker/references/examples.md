# Resume examples

Use these as patterns, not templates.

## before and after example: IaC modernization

Raw:

> I moved services that were manually provisioned into our Ansible IaC project. I also wrote docs so teammates could add new services.

Weak:

> Helped migrate infrastructure to Ansible IaC and wrote documentation.

Rewrite:

> Engineered an Ansible IaC framework that cut cluster deployment time by 80% and enabled teammates to add new services without manual provisioning.

## before and after example: PostgreSQL HA

Raw:

> I set up pgpool for load balancing on our databases. We used to keep making the DB bigger every month. After pgpool we didn't have to.

Rewrite:

> Architected PostgreSQL infrastructure with Pgpool-II load balancing and asynchronous replication, reducing costs by 20% and eliminating monthly vertical scaling while supporting continued user growth.

## before and after example: observability

Raw:

> Deployed Prometheus and Grafana for monitoring. Set up dashboards for our databases and EC2 instances. Incident response got faster.

Rewrite:

> Designed an end-to-end observability platform with Prometheus and Grafana across OS and database metrics, cutting incident response time by 75% and improving proactive system optimization.

## cover letter pattern

Good cover letters:

- open with a specific moment
- connect that moment to the employer's problem
- drop one or two proof points with numbers
- close with a clear next step

Bad cover letters:

- start with "I am excited to apply"
- summarize the whole resume
- use buzzwords instead of evidence

## correction patterns

| weak input | stronger rewrite pattern |
|------------|--------------------------|
| helped with deployment pipelines | built CI/CD pipelines using [tool], resulting in [metric] |
| was responsible for monitoring | deployed and managed observability tooling across [systems], improving [metric] |
| worked on cost optimization | reduced monthly cloud spend by [metric] by implementing [change] |
